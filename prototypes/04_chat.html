What does the payments table track?
Cortex

The payments table records every payment transaction for an order. Each row is one payment installment. A single order can have multiple payment rows if the customer split the payment across methods or installments.

ColumnWhat it stores
order_idLinks to the orders table
payment_sequential1st, 2nd, 3rd payment for same order
payment_typecredit_card, boleto, voucher, debit_card
payment_installmentsNumber of installments chosen
payment_valueAmount paid in BRL

Write me a SQL query to get average delivery time per state
Cortex

Here is the query. It joins orders with customers to get the state, then calculates the average days between purchase and delivery.

SQL
SELECT c.customer_state, COUNT(*) AS total_orders, ROUND(AVG( EXTRACT(EPOCH FROM o.order_delivered_customer_date - o.order_purchase_timestamp ) / 86400 ), 1) AS avg_delivery_days FROM orders o JOIN customers c ON o.customer_id = c.customer_id WHERE o.order_status = 'delivered' AND o.order_delivered_customer_date IS NOT NULL GROUP BY c.customer_state ORDER BY avg_delivery_days DESC;

This filters for delivered orders only and excludes rows where the delivery date is null (3.2% of orders). Results will show all 27 Brazilian states ranked by slowest delivery.
